# Developer & Operations Control Plane

## 目的

システム自身が高スキル人材と手作業のボトルネックにならないよう、少人数で開発、運用、改善、コスト統制を行う。

## 管理ビュー

- Product: 機能要望、重複、利用効果、優先順位
- Engineering: リードタイム、PR待ち、CI失敗、技術負債
- Operations: SLO、エラー、遅延、キュー滞留、同期状態
- AI Quality: 根拠性、完遂率、修正率、権限逸脱
- FinOps: 部門・機能・エージェント・モデル別原価
- Governance: 権限、DLP、ポリシー、例外、監査

## 開発フロー

要望 → AIトリアージ → 人間優先順位 → 仕様/受入条件案 → AI支援実装 → PR → 自動品質ゲート → プレビュー → 人間レビュー → Feature Flag → カナリア → 観測。

## 運用目標

成熟後はプロダクト、開発、SRE、AI品質、M365統制、UXを合計5人相当以下の中核チームで運営可能にする。
