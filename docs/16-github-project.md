# GitHub Project設計

## Project名

OIP Delivery

## フィールド

- Status: Backlog / Ready / In Progress / Review / Blocked / Done
- Priority: P0 / P1 / P2 / P3
- Type: Epic / Feature / Security / Research / Operations / Documentation
- Area: Identity / Context / Decision / Health / Agents / UX / Platform / Security / FinOps
- Phase: 0-5
- Effort: XS / S / M / L / XL
- Risk: Low / Medium / High / Critical
- Target date

## ビュー

1. Roadmap: Phase別のテーブル
2. Delivery Board: Status別カンバン
3. Security: SecurityラベルとRisk
4. Research: Research Issue
5. Executive: Epic、Priority、Targetのみ
6. Operations: SLO、コスト、障害、技術負債

## 運用

- Issueは課題と成果を記載し、解決策だけを書かない
- Readyへ移す前に受入条件と検証方法を必須化
- Blockedには依存Issueと解除条件を記載
- Done後に利用効果、品質、コストを確認する
