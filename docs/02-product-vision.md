# プロダクトビジョン

## 中核体験

社員がTeamsで対話すると、システムは案件、関係者、時系列、過去の決定、適用基準を解決し、論点、仮説、合意候補、未決事項を構造化する。人間が意味と責任を確認すると、企画書、提案書、タスク、KPI、ダッシュボードへ一貫して反映される。

## 4つの利用モード

| モード | AI | 人間 |
|---|---|---|
| Explore | 問い、仮説、視点を広げる | 理想と価値を語る |
| Challenge | 前提、反対意見、失敗要因を提示 | 考えを再検討する |
| Decide | 選択肢、根拠、不確実性を構造化 | 判断し責任を持つ |
| Execute | 成果物、タスク、通知を生成 | 例外と関係性を扱う |

## 主な利用者

- 経営層: 判断キュー、30秒/3分/原本ビュー
- 管理職: 複数案件のヘルス、意思決定待ち、要員負荷
- PM/企画: 合意、成果物、タスク、リスクの統合管理
- 現場: 今日の優先事項、変更影響、必要なアクション
- 専門家: 原本、詳細差分、基準、技術根拠
- 開発・管理者: 品質、セキュリティ、コスト、利用効果
