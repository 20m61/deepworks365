# Security Policy

## Reporting a vulnerability

公開Issueには脆弱性の詳細を記載しないでください。GitHubのPrivate Vulnerability Reportingが有効な場合はそれを使用し、利用できない場合はリポジトリ管理者へ非公開で連絡してください。

報告には以下を含めてください。

- 影響する機能・バージョン
- 再現手順または概念実証
- 想定される影響
- 既知の緩和策

## セキュリティ原則

- Entra ID / Microsoft 365のアクセス権を正とする
- 最小権限、委任アクセス、短命資格情報、OIDCを優先する
- AIはアクセス権、正式版、業務状態を決定しない
- 重要操作はプレビュー、差分、人間承認、監査ログを必須とする
- 未信頼入力をエージェントの命令として扱わない
- データ分類、目的制限、保持期間、外部送信制御を実装する
- セキュリティ例外には期限・責任者・代替統制を設定する

詳細は [セキュリティアーキテクチャ](docs/12-security-architecture.md) を参照してください。
